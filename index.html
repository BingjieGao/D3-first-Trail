Data Cleaning

Data Validation

The mixed data format of date like the first picture image shows. The main format of expressing the date in the form is "dd/mm/yyyy" but in the selection area is "yyyy-mm-dd".

This mix use of different formats could be modified with the automatic transform with the cells in the certain column like the picture illustrated with the function "to date". It will automatically format the date into a united layout

Multiple Representations

There are multiple ways of presenting the same department like the picture shows in the red rectangle which the department of agriculture is stored differently. The issue would simply cause the redundant of data.

In Google Refine it is easy to modify the multiple representations with the cluster. In the image here, it could be seen that with certain method and distance function, the redundant representations could be eliminated automatically.

Summation Records

The summation Errors are normal for redundant data presentation that each key term would be sum to a total like the picture showed. This "Total" line would easily cause the diffculty visualizing the main data as well as the information presentation.

The summaiton records are all deleted for a better vision while processing the data as well as presenting them.

Mixed use of numerical scales

The unit of each numerical terms could be mistaken as in the picture states. The number in the red rectangle clearly illustrates that the units between the numbers belong to each line are different: "8127" and "8.127". It could be difficult for data analysts to identify which exact numerical experession is not correct consider its' unit and cause certain illusion.

In order to make sure that no extra data errors are to be produced as to modify certain numerical scales without knowing the actual value. The way of correcting this was to delete the row contains such mix use of numerical errors

Spelling Errors

This is another form of the mistaken record of data as it should be "Agriculture" but with an apparent spelling error as the figure shows. Again, the inconvenience of analyse and collect data to real and useful information could be contributed with this input mistake.

Wrong lines

It is clear as the figure presented that the data records were shifted to right by certain columns which makes the data incorrectly and meaningless. Excel provides the easiest way of moving the columns to where it belongs to match the titles.

Visualization 1

Description

This single Line group was designed for observing the “LifeCycle Cost” of each Department through the year. So that the trend as well as the certain detailed "LifeCycle Cost" could be presented with hover.

Visualization

Click on each radio button representing each different Department, the single line describes this corresponding "LifeCycle Cost" of this Department would be the only one left.

The “LIfeCycle Cost” of each Department could be clearly presented and analysed regarding the increasing or decreasing trend. In this case, the target user would be those who care about the actual cost of different Departments throughout each year in order to understand the basic trend as well as the situations across the time.

Visualizaiton 2

Description

In the first part of this graph the “LifeCycle Cost” of each Department was displayed in bar chart. While correspondingly the second part of this graph, the Scatter chart would have certain interactions according to the “year” selected from the Bar chart. The Scatter Chart presents the comparison between "Planned Cost" and the "Actual Cost" of each project under certain Department.

Visualization

The interactions between the legend in the middle and both of the Scatter chart, and Bar chart are syncronized in order to display a different perspective according to the Department selected. By Clicking on each small rectangle both parts of the graph interactes, while clicking on the presented bar, the "scatters" would show the balance specificly in the Department in the year chosen. In this case, the user would be more concentrated on concerning the balance between planned cost and actual/project cost in order to observer which exact project under the specific Department exceeded the budget significantly compared with what was planned correspondingly with the year selected from the Bar chart.
